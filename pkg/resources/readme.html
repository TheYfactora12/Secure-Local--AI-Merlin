Merlin AI — First Run

Your private AI. On your Mac. Forever.

Start in the Merlin Dashboard, then run the doctor check. Cloud/API providers are optional and disabled by default.

1. Open Merlin Dashboard

http://localhost:8888

The Merlin Dashboard is the read-only Merlin AI command center for local chat, local service readiness, privacy status, model routing, memory, approvals, and system health.

2. Check local readiness

cd ~/merlin-ai
bash scripts/doctor.sh
wizard merlin status

Merlin should report local-first mode, service health, and any degraded startup state honestly. Do not treat warnings as ready until reviewed.

3. Optional: add local models or API keys

# Local model pull is explicit
bash scripts/add-model.sh qwen2.5:7b

# Optional cloud/API keys only if you choose to enable them later
nano .env

The package installer uses the core path with model pulls skipped by default. No cloud calls are enabled unless you configure them explicitly.

Local service map

  • Merlin Dashboard — http://localhost:8888
  • Open WebUI — http://localhost:3000
  • Merlin Status API — http://localhost:8765
  • Merlin Task API — http://localhost:8766
  • LiteLLM — http://localhost:4000
  • Qdrant Dashboard — http://localhost:6333/dashboard
  • SearXNG — http://localhost:8080 when the search profile is enabled
  • Perplexica Search — http://localhost:3002 when the search profile is enabled
  • n8n Automation — http://localhost:5678 when the automation profile is enabled

Useful commands

# Open Merlin Dashboard
wizard open

# Check all services
bash ~/merlin-ai/scripts/doctor.sh

# Safe upgrade (with auto-rollback)
bash ~/merlin-ai/scripts/upgrade.sh

# Stop everything
bash ~/merlin-ai/scripts/stop.sh

# Uninstall
bash ~/merlin-ai/pkg/scripts/uninstall.sh

# Full Merlin purge before a clean reinstall test
bash ~/merlin-ai/pkg/scripts/uninstall.sh --purge-all

Troubleshooting
